Cancer Detection in Pathological Images

Using a pathology data set equipped with over 100,000+ images of cancerous and non-cancerous patches of cells, a model was created to figure out if cancer is present in a pathological scan.

Keras

Using the Keras libaray in Python, convolutional neural networks were built with ease and with high accuracy

Kaggle

All the data used for this project came from Kaggle; a data science forum. In particular the Histopathologic Cancer Detection Dataset was used.

Python

Everything coded in Python as it is the most common tool for Data Science. Other helper libaries were also used when creating the model.

The Process

How the machine learning model was built.

1

Data Collection

The data used for this model was the Histopathologic Cancer Detection dataset on Kaggle. It contains over 100 000 + images of cancerous and non-cancerous cells. Each image is labelled with either a 1, indicating cancer within the image or a 0, indicating no cancer. It was trained on a convolutional neural network so that new images could be readily identified as cancerous or non-cancerous.

2

Model Architeture + Training

The model architeture for this model consists of 6 convolutional layers and 5 dense layers. The convolutional layers perform feature extraction from the images, condensing it into easily transferrable detection. The dense layers take in the features of the image and determine of cancer is within the image. Droupout layers were used to ensure that the model didn't memorize the dataset and generalized to all images of pathological cancer scans. The model was trained for 10 hours over 20 epochs.

3

Results

The model was able to achieve a 80% accuracy of detection rate for any new image containing cancerous cells. This is a good system for preliminary tests and further determining if someone has cancer. The future steps for this project include training the model for a longer time to increase the accuracy and start using biomarkers as a way to determine if someone has cancer. Also, increasing the amount of data could yield better results.
